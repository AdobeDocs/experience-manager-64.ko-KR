## AEM 6.4.8.0에 포함된 기능 {#what-s-included-in-aem}

AEM 6.4.8.0은 AEM 6.4의 공식 출시 이후 릴리스된 새로운 기능, 주요 고객이 요청한 향상된 기능 및 성능, 안정성, 보안 개선 사항이 포함된 중요한 업데이트입니다 **2018년 4월**

또한 누적되므로 6.4.8.0에는 이전에 릴리스된 모든 AEM 6.4 서비스 팩이 포함됩니다.

이 서비스 팩 릴리스의 몇 가지 주요 사항은 다음과 같습니다.

* 내장된 저장소(Apache Jackrabbit Oak)가 버전 1.8.20으로 업데이트되었습니다.

* WCM-RTE의 일본어 웹 사이트에 대해 자동 줄 바꿈 기능이 지원됩니다.

* 일본어 웹 사이트에서는 단어 브레이크 및 줄 바꿈 처리가 지원됩니다.

* 적절한 위치에서 일본어 문장과 행을 중단하기 위해 BUNSETSU 방법을 사용하는 지원이 추가되었습니다.

* 서신 관리를 위한 CCR UI는 이제 독일어 로케일에 대한 십진수 값을 지원합니다.

* 이제 SOAP 웹 서비스를 사용하여 양식 데이터 모델 통합을 통해 요소에 대한 선택 그룹 또는 속성을 지원합니다.

* 이제 AEM Assets이 을 통해 Brand Portal으로 구성됩니다 [!DNL Adobe I/O].

* ContextHub에 번들로 포함된 jQuery 버전을 3.2.1로 업데이트했습니다.

## 변경 사항 목록 {#list-of-changes}

### Sites {#sites}

* AEM Sites 페이지의 URL에 콜론 또는 백분율 기호가 포함되어 있으면 기본 브라우저가 응답을 중단하고 CPU 사이클에 스파이크가 표시됩니다(NPR-32368, NPR-31917).
* AEM Sites 페이지를 열어 편집하고 구성 요소를 복사하면 일부 자리 표시자에 대해 붙여넣기 작업을 사용할 수 없습니다(NPR-32328).
* 활성화 요청 워크플로우에 참조된 자산이 포함되지 않습니다(NPR-32304).
* 블루프린트를 만들 때 레코드 수가 80개를 초과하는 경우 처음 80개의 레코드만 표시됩니다. 블루프린트는 나머지 레코드에 대한 빈 라인을 표시합니다(NPR-32058).
* 사용자는 필수 필드에 정보를 제공하지 않고 컨텐츠 조각을 저장할 수 있습니다(NPR-31988).
* 코어 경험 조각 구성 요소에 구성된 경로에 대해 자동 탐색이 작동하지 않습니다(NPR-31921).
* 리치 텍스트 편집기(RTE)에서 표 셀의 유형을 변경하면 아래 오류가 발생합니다.
   `Error: No common ancestor found, cannot continue` (NPR-31916).
* 동일한 폴더 내에서 컨텐츠를 이동하면 페이지 이동 옵션이 비활성화됩니다(NPR-31841).
* BUNSETSU 메서드를 사용하여 일본어 문장을 적절한 위치에서 분할하는 지원이 추가되었습니다(NPR-31836).
* 리치 텍스트 편집기(RTE)에서 하이퍼링크를 편집하면 새로 선택한 경로가 저장되지 않습니다(NPR-31659).
* 다중 필드 구성 요소를 삭제하고 삭제를 취소하면 구성 요소가 복원되지만 데이터가 복원되지 않습니다(NPR-31617).

### Assets {#assets}

* Dynamic Media Classic 구성을 사용하여 Experience Manager에서 자산을 폴더 간에 이동하는 동안 이름이 없는 폴더가 Dynamic Media Classic에 생성됩니다(NPR-32440).

* Dynamic Media Scene7 모드에서 실행되는 Experience Manager에 PDF 파일의 자산 세부 사항 페이지에 작업 버튼이 표시되지 않습니다(NPR-32316).

* 자산 및 비디오 표현물은 삭제할 수 없습니다(NPR-32213).

* 활성화가 예약된 달력 아이콘이 이후 날짜 및 시간에 대해 예약된 자산의 상태 열(DAM 자산 목록의 클래식 UI에 있음)에 표시되지 않습니다(NPR-32198).

* 자산이 다른 자산을 사용하여 두 개 이상의 자산과 관련되면 자산 관계를 덮어씁니다(NPR-32196).

* 사용자가 Dynamic Media 클라이언트의 설정 편집기에서 변경하지 않은 경우 저장 버튼이 원격 세트를 가져오지 않습니다(NPR-32178).

* PSD 자산 섭취로 CPU 스파이크와 응답하지 않는 Experience Manager 작성자 인스턴스가 발생합니다(NPR-32165).

* 대용량 ZIP 파일이 Experience Manager DAM에 업로드될 때 관찰되는 메모리 부족 예외(NPR-32155).

* 버전 기록 URL은 자산 속성 페이지의 참조자 필드 아래에 표시됩니다(NPR-31889).

* 게시 관리 페이지의 Brand Portal에서 게시 취소가 게시된 폴더의 하위 폴더에 대해 실패합니다(NPR-31835).

* Scene7 Cloud 구성이 비공개 폴더에 배치되면 Dynamic Media 비디오 인코딩이 업로드되지 않습니다 `/conf` 대신 `/conf/global` (NPR-31779).

* Dynamic Media Scene7 실행 모드에서 실행되는 Experience Manager에서 주석이 추가된 후 타임라인에 이미지가 표시되지 않습니다(NPR-31754).

* DAM에서 다운로드한 ZIP 파일은 WinZip을 사용하여 열 수 없습니다(NPR-31745).

### 통합 {#integrations-6480}

* 다음 **회사** 및 **보고** 세트 드롭다운 메뉴는 한 번 숨겨집니다 **보고 소스** Experience Manager 클라우드 서비스에서 Adobe Analytics을 구성하는 동안 이 선택되었습니다(NPR-31729).

* ReportSuitesServlet이 SSRF에 취약합니다(NPR-32161).

### 슬링 {#sling-6480}

* 리소스 관찰의 비결정적 표시가 작동하지 않습니다(CQ-4286466).

* 사용자에게 하위 폴더에서 프로젝트를 만들 수 있는 권한이 있어도 만들기 버튼이 표시되지 않습니다(NPR-31831).

### WCM-MSM {#wcm-msm-6480}

* 권한 문제로 인해 블루프린트 내에서 페이지를 이동하는 동안 오류가 표시됩니다(NPR-32610).

* 문서 서비스: XDP 파일을 PDF 파일로 어셈블하고 변환하는 동안 오류 메시지가 표시됩니다(NPR-31821).

* Analytics는 사이트 페이지에서 양식 제출 또는 포기 시 적절한 결과를 표시하지 않습니다(NPR-31359).

### 이전 서비스 팩에 포함된 핫픽스 및 기능 팩 {#hotfixes-and-feature-packs-included-in-previous-service-packs}

#### AEM 6.4.7.0 {#experience-manager-6470}

AEM 6.4.7.0은 AEM 6.4의 공식 출시 이후에 발표된 성능, 안정성, 보안 및 주요 고객 수정 사항과 개선 사항을 포함하는 중요한 업데이트입니다 **2018년 4월**

또한 누적 사항이므로 6.4.7.0에는 모든 AEM 6.4 서비스 팩 릴리스 이전에 포함되어 있습니다.

AEM 6.4.7.0의 주요 특징 중 일부는 다음과 같습니다.

* 내장된 저장소(Apache Jackrabbit Oak)가 버전 1.8.17으로 업데이트되었습니다.
* 삭제하는 동안 사이트 페이지 버전을 설정하는 지원이 추가되었습니다.
* 정렬 가능한 새로 만든 날짜에 대한 열이 추가되었습니다. **DAM 목록** 자산 검색 결과 보기 및 설정 **목록** 보기(NPR-31311).
* 자산 정렬 기준 **이름** 열은 **목록** 보기.
* 이제 Dynamic Media의 UI에서 재처리 및 배치 업로드에 대한 배치 크기 및 워크플로우 단계 시간 제한을 구성할 수 있습니다.
* 다음 `pdfBrochure` IPS에 메모리를 저장하도록 Scene 7 클라우드 구성에서 false로 설정되어 있습니다.

##### 자산 {#assets-6470}

**제품 개선 사항**

* API 패키지의 내보내기 버전 `package com.day.cq.dam.handler.standard.msoffice` 지원 `dam-handler` 번들이 6.0.0으로 업그레이드되었습니다(CQ-4279059).
패키지를 사용하는 경우 `com.day.cq.dam.handler.standard.msoffice` 사용자 지정 구현에서 다음을 컴파일하는 것이 좋습니다 `dam-handler` 최신 uber jar로 번들입니다.

* 정렬 가능한 새로 만든 날짜에 대한 열이 목록 보기의 DAM 목록 보기 및 자산 검색 결과에 추가되었습니다(NPR-31311).

* 이름 열을 기반으로 한 자산 정렬이 목록 보기에서 허용되었습니다(NPR-31299).

**수정 사항**

* 일부 PDF 문서의 메타데이터는 해당 제목을 수정할 때 업데이트되지 않고 PDF에 저장됩니다(NPR-31575).

* 파일 이름에 &#39;+&#39; 기호가 있는 자산은 삭제할 수 없습니다(NPR-30588).

* DAM 폴더 속성은 닫힌 사용자 그룹에 추가된 사용자 또는 그룹(LDAP 동기화로 생성)을 표시하지 않습니다(NPR-30555).

* 이메일 템플릿 제목란에서 발생하는 특수 문자가 제대로 표시되지 않습니다(NPR-30547).

* Dynamic Media Scene 7 런타임 모드에서 실행 중인 AEM에서 자산을 한 폴더에서 다른 폴더로 이동할 때 자산 이름이 소문자로 변경됩니다(NPR-31631).

* 이미지 집합(또는 미디어 집합)을 만들고 DAM에서 적절한 명명 규칙을 사용하여 이름을 지정하면 Scene 7에서 이미지 집합 이름이 소문자로 변경됩니다(NPR-31576).

* Dynamic Media 인코딩 비디오 워크플로우가 Scene 7에서 Dynamic Media - Scene 7 실행 모드로 마이그레이션된 비디오의 축소판을 생성하지 못합니다(NPR-31523).

* Dynamic Media - Scene 7 런타임 모드에서 실행되는 AEM에서 필터를 사용하여 세트를 검색하는 동안 내부 서버 오류가 관찰되었습니다(NPR-31388).

* 원격 이미지 집합을 편집하는 동안 Scene7 회사 이름과 같은 폴더에 있는 이미지에 대한 오류가 발견되었습니다(NPR-31347).

* 참조가 포함된 자산이 게시되지 않습니다(DM)(NPR-31179).

* Dynamic Media 하이브리드 모드에 대해 구성된 만료(클라이언트 캐시 TTL(Time To Live)) 값이 Dynamic Media 게재 환경(NPR-31126)에 복제되지 않습니다.

* AEM Dynamic Media - Scene 7 런타임 모드에서 Scene 7로의 업로드를 완료하는 데 너무 오래 걸립니다(NPR-30926).

#### AEM 6.4.5.0 {#experience-manager-6450}

AEM 6.4.5.0은 AEM 6.4의 공식 출시 이후에 발표된 성능, 안정성, 보안 및 주요 고객 수정 사항과 개선 사항을 포함하는 중요한 업데이트입니다 **2018년 4월**

#### AEM 6.4.4.0 {#experience-manager-6440}

AEM 6.4.4.0은 AEM 6.4의 공식 출시 이후에 발표된 성능, 안정성, 보안 및 주요 고객 수정 사항과 개선 사항을 포함하는 중요한 업데이트입니다 **2018년 4월**

또한 누적 사항이므로 6.4.4.0에는 모든 AEM 6.4 서비스 팩 릴리스 이전에 포함되어 있습니다.

AEM 6.4.4.0의 주요 특징 중 일부는 다음과 같습니다.

* 내장된 저장소(Apache Jackrabbit Oak)가 버전 1.8.11으로 업데이트되었습니다.
* 서비스 버전 HTTP 요청이 자주 발생하지 않도록 서비스 버전 캐싱 지원을 추가했습니다.
* 자동 태그 삭제에 대한 지원이 추가되었습니다.
* 카탈로그 마법사에 대한 무한 스크롤을 구현했습니다.
* 커뮤니티 사이트에 따라 권한을 제한하는 기능을 지원합니다.
* Sling Granite 컨텐츠 상태 확인에 대한 성능 수정(캐싱, 비동기 실행, 제외 목록).
* pagethumboqunail 대화 상자에 자산 선택기 단추가 추가되었습니다.
* 필드에 도구 설명 위치를 지정할 수 있도록 새 속성을 추가했습니다.
* Multifield 내에서 ColorPicker 지원을 개선했습니다.
* 컨텐츠 조각의 clientlibs에서 숫자 입력 다중 필드에 대한 빈 값을 무시하기 위한 검사를 추가했습니다.
* Microsoft Translator 텍스트 API v3에 대한 지원을 사용하도록 설정되었습니다.

#### AEM 6.4.3.0 {#experience-manager-6430}

AEM 6.4.3.0은 2018년 4월에 AEM 6.4의 공식 출시 이후에 발표된 성능, 안정성, 보안 및 주요 고객 수정 사항과 개선 사항을 포함하는 중요한 업데이트입니다.

또한 누적 사항이므로 6.4.3.0에는 모든 AEM 6.4 서비스 팩 릴리스 이전에 포함되어 있습니다.

AEM 6.4.3.0의 주요 특징 중 일부는 다음과 같습니다.

* 내장된 저장소(Apache Jackrabbit Oak)가 버전 1.8.9으로 업데이트되었습니다.
* 적응형 양식 템플릿에 allowedPaths 속성에 대한 지원이 추가되었습니다.
* AEM에서 자산에 대한 향상된 패널 기반 검색
* 로그인 페이지의 XSS(교차 사이트 스크립팅) 수정 사항.
* UI 계기가 개선되었습니다.
* FormData 처리 개선.
* Multifield 내에서 항목 이름 지정 처리가 개선되었습니다.
* 선택 중에 자리 표시자 항목(카드 보기 및 목록 보기)에 대한 처리가 개선되었습니다.
* Managed Services에 대한 Adobe IMS 인증 및 Admin Console 지원이 추가되었습니다.

#### AEM 6.4.2.0 {#experience-manager-6420}

AEM 6.4.2.0은 AEM 6.4의 공식 출시 이후에 발표된 성능, 안정성, 보안 및 주요 고객 수정 사항과 개선 사항을 포함하는 중요한 업데이트입니다 **2018년 4월**
또한 누적 사항이므로 6.4.2.0에는 모든 AEM 6.4 서비스 팩 릴리스 이전에 포함되어 있습니다.

AEM 6.4.2.0의 주요 특징 중 일부는 다음과 같습니다.

* 내장된 저장소(Apache Jackrabbit Oak)가 버전 1.8.7으로 업데이트되었습니다.
* HTL(HTML 템플릿 언어) 사양 1.4 기능에 대한 지원이 추가되었습니다
* MongoDB Enterprise 3.6에 대한 지원이 추가되었습니다.
* 사이트 페이지 편집기는 컨텍스트 내 편집 및 컴포지션에 대한 지원을 추가하며 React 또는 Analysis Workspace와 함께 클라이언트 측 구성 요소를 빌드합니다 <a href="../sites-developing/spa-walkthrough.md">AEM SPA Editor JS SDK</a>.
* 컨텐츠 조각 개선 사항: 텍스트 필드에 주석을 달고, 버전을 나란히 비교하는 기능을 추가했습니다.
* 추가됨 [Adobe Stock과 통합](/help/assets/aem-assets-adobe-stock.md) AEM 사용자 인터페이스에서 직접 Adobe Stock 자산을 검색, 미리 보기, 저장 및 라이선스를 제공할 수 있습니다. 자세한 내용은 [AEM Assets에서 Adobe Stock 자산 사용](https://docs.adobe.com/content/help/en/experience-manager-learn/assets/creative-workflows/adobe-stock.html).
* 자산은 동적 조건부 메타스키마 및 자산 폴더에 대한 메타데이터 스키마를 설정하는 기능에 대한 지원을 추가했습니다.
* 폴더 축소판 생성/업데이트 기능을 활성화/비활성화하는 구성이 각 구성 요소에 추가되었습니다.
* 페이지 작성 시 이미지 편집기 개선 사항.
* 선택한 응답이 삭제되는 경우 점수 이벤트를 위한 커뮤니티의 회귀 수정 사항이 제대로 처리되지 않습니다.

* 트랜잭션 초기화 작업은 storeTransaction에 대한 첫 번째 호출에서만 시작됩니다.

#### AEM 6.4.1.0 {#experience-manager-6410}

AEM 6.4.1.0은 2018년 4월에 AEM 6.4의 공식 출시 이후에 발표된 성능, 안정성, 보안 및 주요 고객 수정 사항과 개선 사항을 포함하는 중요한 업데이트입니다.

AEM 6.4.1.0은 AEM 6.4 GA에 설치할 수 있습니다. 서비스 팩의 주요 특징 중 일부는 다음과 같습니다.

* 내장된 저장소(Apache Jackrabbit Oak)가 버전 1.8.3으로 업데이트되었습니다.
* 향상된 스마트 태그가 도입되었습니다.
* 디자인 선택기의 수정 사항, 태그 선택기(소스 VM 및 대상 VM을 자동으로 변경)
* 값을 문자열로 저장하도록 typeHint 지원 추가
* 메일 서비스에서 TLS를 통해 SMTP에 대한 지원을 추가했습니다.
* DMS7에서 HTTP 전달자에 대한 프록시 처리 수정 사항입니다.
* /etc/clientlibs/social/thirdparty/handlebars/source/handlebars.js 버전 1.3으로 업데이트합니다.
* DMHybrid 옵트아웃 및 옵트인 패키지의 수정 사항.
* 스마트 자르기의 수정 사항.
* OOTB 구성 값을 새 위치( /etc에서 /conf로)로 마이그레이션했습니다.
* 게재 서버의 고객 설정에 색상 프로필을 추가했습니다.
* /etc —&amp;gt;에서 이미지 서버 설정 마이그레이션 /conf.
* 번역을 위한 소스 컨텐츠 조각을 추가했습니다.
* 인쇄 및 인쇄 대화 상자에 ARIA 지원이 추가되었습니다.
* 이메일 유효성 검사 ARIA 지원이 추가되었습니다.
* platform.clientlibs 수정 사항에 대한 사전 예방적 백포트입니다.
* 명시적 dataType에 입력이 없을 때 스크립트가 자동으로 실행되지 않도록 합니다(CVE-2015-9251 확인).

#### NPR-24536에 필요한 구성 설정 {#configuration-settings-required-for-npr-1}

사용자가 작업을 요청할 때 다른 사용자에 대한 공유 큐 개수는 기본적으로 새로 고쳐지지 않습니다. 이 문제를 해결하기 위해 새로운 속성을 도입했습니다. 아래 절차에 따라 AEM 인스턴스에서 이 속성을 구성하십시오.

1. 관리자 UI -> 서비스 -> 작업 공간 -> 전역 관리로 이동합니다.
1. 전역 설정을 내보냅니다.
1. 다운로드한 XML 파일에서 태그를 추가합니다 &lt;client_taskspollinginterval>10&lt;/client_taskspollinginterval> 여기서 10은 초 단위의 예제 값입니다. 그에 따라 수정할 수 있습니다.
1. 파일을 저장합니다.
1. 관리자 UI -> 서비스 -> 작업 공간 -> 전역 관리로 돌아갑니다.
1. 전역 설정 가져오기 섹션에서 xml 파일을 가져옵니다.
1. 이제 시스템에서 로그아웃한 다음 다시 로그인할 수 있습니다.
1. 이제 작업 공간의 다른 사용자에 대한 공유 큐 개수를 새로 고칩니다.
1. 폴링을 해제하려면 값을 0으로 변경하고 XML 파일을 다시 가져옵니다.
